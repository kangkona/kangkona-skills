# Kangkona Skills

> Reusable builder playbooks for sharper decisions and credible shipping under real constraints.

Languages: English | [中文](README.zh-CN.md)

## Install

Install the full skill collection:

```bash
npx skills add kangkona/kangkona-skills
```

Then invoke a playbook from a skills-enabled assistant:

```text
Use $hackathon to choose a hackathon project, sharpen the narrative, plan execution, and prepare the demo, deck, and Q&A.

Use $strategic-naming to create or evaluate product, company, feature, campaign, or brand-system names with a strategy-first naming process.
```

## Skills Index

| Skill | Description |
| --- | --- |
| [hackathon](skills/hackathon) | Battle-tested hackathon strategy for choosing rule-compliant projects, shaping the narrative, building the minimum credible demo, preparing slides/video/submission materials, and handling judge Q&A. |
| [strategic-naming](skills/strategic-naming) | Lexicon-inspired strategic naming workflow for product, company, feature, campaign, and brand architecture work: define the naming thesis, build lanes, generate broadly, screen with sound/meaning/risk, and validate domains/collisions before choosing. |

## Why Use These Skills

- **Battle-tested**: distilled from repeated hackathon and builder-side outcomes, not abstract advice.
- **Scoreboard-first**: starts from the rubric, deadline, audience, track, and deliverables before picking a project.
- **Rule-aware**: explicitly checks originality, pre-existing code, external dependencies, APIs, datasets, and submission constraints.
- **Demo-oriented**: optimizes for one visible, credible, end-to-end loop instead of a fragile pile of features.
- **Bilingual**: written to support both English and Chinese collaboration.

## Current Collection

### Hackathon

Use this skill when you need to:

- Choose among multiple hackathon project ideas.
- Turn a fuzzy idea into a clear one-liner, category, and demo loop.
- Run a portfolio race across several candidate projects without getting attached too early.
- Build a minimum credible demo with screenshots, fallback paths, and hard-to-fake proof.
- Prepare a short pitch, Q&A defense pack, and final-hour submission checklist.

### Strategic Naming

Use this skill when you need to:

- Name or rename products, companies, features, campaigns, AI systems, or internal tools.
- Diagnose why candidate names feel wrong, generic, hard to own, or culturally off.
- Build naming lanes, compare shortlists, and choose a canonical domain and brand architecture.
- Create a launch-ready naming package with positioning line, risks, and next validation steps.
